## المرحلة 3 — الربط بقاعدة بيانات حقيقية (Supabase) ☁️
**الهدف:** التحول من نموذج تفاعلي إلى منتج متعدد المستخدمين. · **المدة:** 2–4 أسابيع
**المتطلب الوحيد منكِ:** إنشاء حساب Supabase مجاني وتزويدي بمفاتيح المشروع (URL + anon key).

- [ ] إنشاء المشروع وتشغيل الهجرات الجاهزة بالترتيب: `0001` (طاقم/جداول/إجازات/عمولات + RLS) → `0002` (محرك التوفر وحارس الحجوزات) → `0003` (أرصدة، حضور، رواتب متعددة المصادر، تصفية الشهر) → `0004` (الصفحات العامة والروابط المخصصة)
- [ ] **المصادقة**: بريد/جوال OTP عبر Supabase Auth — أدوار: مالكة، موظفة، عميلة
- [ ] **استراتيجية التحويل التدريجي** (Strangler): نبدأ بأعلى قيمة —
  1. الصفحة العامة والحجوزات (`booking.html` → `get_public_page` + `get_available_booking_slots`)
  2. الطاقم والموارد البشرية (الشاشات الحالية تستدعي الـ API بدل localStorage)
  3. الرواتب والتصفية → قسائم PDF من `payroll_slips`
  4. المتجر والتقييمات (تعليقات وصور حقيقية مشتركة بين الزوار)
- [ ] **ترحيل البيانات**: سكربت يقرأ بيانات localStorage الحالية (طاقم، عميلات، إعدادات الصفحة) ويرفعها لحساب المالكة أول تسجيل دخول
- [ ] نشر واجهة Next.js (المكونات جاهزة في `luma-hr/`) على نفس مشروع Vercel تحت `/app`، مع بقاء الموقع التسويقي الثابت كما هو

**معيار الإنجاز:** حجز من جوال عميلة يظهر فوراً على لوحة المالكة من جهاز آخر.
